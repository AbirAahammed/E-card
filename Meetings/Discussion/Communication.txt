[28-Jan-21 11:54 AM] BiPolarPolarBear#9804
**Extreme Programming**
Incremental, doesn't continue until all tests pass. Very quick increments.

**User Stories:**
User stories should have acceptance tests that check they have been completed.
Usually done by automated testing such as DevOps

**Refactoring:**
Don't right shitty code, name stuff properly, create methods for duplicate code, and use hierarchies when necessary.

**Test-Driven Development**
Create test -> Write code until the test passes -> refactor code.
Good practice to involve customers in the process.
Difficult to judge completeness of tests, especially before code is written.
Does increase development time required.

**Pair Programming**
Programmers develop code together.
One codes, one watches while suggesting solutions.
Pairs are changed frequently.
Developers end up knowing much more about the code they work with.
Serves as an informal review.
Reduces project risks, still fairly efficient.

**Scrum**
Objectives -> Incremental sprints -> finish and complete documentation.
Short daily meetings (Scrums)
Usually iterate in 2-4 weeks.
When a sprint is commited, they won't change the plan.
Good for management, everyone is kept up to date on the project.
